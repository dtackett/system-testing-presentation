System Tests

Preamble

We aim to be Highly Aligned, Loosely Coupled Organization

Align on the What, How, Why, Where, When and Who of System Tests

What

  • Exercise requirements of the system like an end user
  • Must be runnable in an environment that mirrors production
  • Test the interaction of the entire system

How

  • With the code base but not part of the software
  • Constructed to test an arbitrary server
  • Strong preference for Selenium ecosystem

Why

  • Once automated retesting is extremely simple
  • Get QA involved earlier in the project
  • Build alignment between how developers are building software and how QA tests the software

Where

  • Convienent to run on developer machines but not sufficient
  • Must run in environments that mirror production
  • Totem will automate running these as part of the flow

When

  • As soon as possible
  • Bringing visibility to defects earlier means addressing them earlier

Who

  • Objective of the team as the team gets the value
  • QA will drive test creation
  • Development should do what they can to help out QA

Already Started

  • Truett's presentation on how System Tests were implemented on the iOS Limited Familiarity App
  • Adam and Alex's presentation on how System Tests were implemented on the Valcyte survey

Important Things to Remember

  • Don't put blind faith in automated tests
  • Individual machines and test environments are not production
